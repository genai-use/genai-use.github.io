# How Usable Are "AI Agent Orchestration" Frameworks?

This is a review of multiple AI agent orchestration frameworks using the Cognitive Dimensions of Notations (CDN) framework. This is intended to fuel discussion around the design and redesign of agent frameworks to enhance their usability.

## Context

You're creating an application that makes a call to Claude's LLM models. To make that API call, you're presented with a few options:

1. Use built-in HTTP methods like python's `requests` or javascript's `fetch`.
2. Install Claude's minimal SDK (software development kit) in your preferred language.
3. Install Claude's Agent SDK, which will abstract your API call as an "agent".
4. Use an AI agent orchestration framework.

I believe most people are divided between Options 2-4, as it doesn't make much sense to go as _low_ as Option 1.

So, the question becomes: which option do you choose?

## Why Usability Matters

One of the most important factors when making a decision related to a library or framework is usability. Defined by Jakob Nielsen, usability is "how easy user interfaces are to use". [^1]

In general, if something is really usable, then it's really easy to use!

Usability is a make or break factor when it comes to software languages and packages. Chances are, when you were looking at the options in the previous section, you were thinking about the simplest (or easiest) option!

Usability is also well-researched, with different methods (like Nielsen's 10 Usability Heauristics) that help us design usable systems.

## Usability Changes Based on the Application

A hammer is pretty good at hammering things in. But, it's not so good at screwing stuff.

In the same way, Options 2-4 depend on how will they be used. Are you developing a prototype? A simple AI assistant, or are you building a complex tool with a larger team?

The _type_ of application you're building matters.

## Alright, alright... How Do We "Grade" Usability?

There's many different options out there. I happen to pick the Cognitive Dimensions of Notations (CDN) framework because it's:

- Simple
- Easy to Understand
- Well-Known
- Previously Applied to Programming Languages
